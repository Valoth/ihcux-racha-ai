# Sobre o Projeto

O RachaAí é um protótipo de baixa fidelidade desenvolvido para a disciplina de Interação Humano Computador e UX.

O objetivo do projeto é simplificar a divisão de contas coletivas entre pessoas que compartilham moradia, viagens ou eventos em grupo, reduzindo conflitos relacionados a pagamentos, cobranças e organização financeira.

O protótipo foi desenvolvido utilizando o Miro, com foco em clareza visual, facilidade de navegação e transparência das informações.

---

# Problema Identificado

Dividir contas de aluguel, internet, energia, mercado e lazer frequentemente gera problemas em grupos que compartilham despesas.

Entre os principais problemas observados:

- Esquecimento de pagamentos
- Falta de transparência
- Dificuldade para calcular divisões
- Cobranças desconfortáveis
- Erros em valores compartilhados

O aplicativo busca reduzir esses atritos através de uma interface simples e objetiva.

---

# Persona

O projeto foi pensado principalmente para:

- Estudantes universitários
- Pessoas que dividem apartamento
- Repúblicas estudantis
- Grupos de viagem
- Amigos que compartilham despesas recorrentes

Esses usuários geralmente precisam organizar pagamentos de maneira rápida e clara no dia a dia.

---

# Heurística de Nielsen Priorizada

## Visibilidade do Status do Sistema

A principal heurística aplicada no protótipo foi a Visibilidade do Status do Sistema.

O aplicativo procura informar constantemente:

- Quem pagou
- Quem ainda deve
- Valores pendentes
- Confirmações de pagamento
- Situação geral do grupo

O objetivo é reduzir ansiedade e aumentar a sensação de segurança durante o uso do sistema.

# Funcionalidades do Protótipo

## Dashboard de Grupos

Tela inicial contendo:

- Lista de grupos ativos
- Saldo geral consolidado
- Resumo rápido de dívidas
- Status simplificado dos grupos

---

## Detalhes do Grupo

Tela com:

- Histórico de gastos
- Informações de pagamento
- Tipo de divisão utilizado
- Participantes envolvidos
- Situação financeira do grupo

---

## Adicionar Gasto

Fluxo para cadastrar novas despesas contendo:

- Descrição
- Valor
- Categoria
- Quem realizou o pagamento
- Método de divisão

---

## Scanner de Nota Fiscal

Tela conceitual para leitura de notas fiscais utilizando câmera.

Objetivo:

- Facilitar inserção automática de valores
- Reduzir erros manuais
- Melhorar velocidade de cadastro

---

## Extrato de Dívidas

Tela responsável por exibir:

- Quem deve para quem
- Valores pendentes
- Total consolidado
- Botão de pagamento

---

## Confirmação de Pagamento

Tela de feedback após quitação de dívida.

Objetivo:

- Confirmar ação do usuário
- Reduzir incerteza
- Transmitir segurança

# Fluxo de Tarefa

## Cadastro de Conta de Luz Dividida Entre 3 Pessoas

1. O usuário acessa o Dashboard.
2. Seleciona o grupo "Casa".
3. Clica em "Adicionar Gasto".
4. Insere:
   - Descrição da conta
   - Valor da conta
   - Quem realizou o pagamento
5. Seleciona "Divisão Igual".
6. O sistema calcula automaticamente os valores.
7. O gasto é registrado no grupo.
8. O extrato é atualizado mostrando quem deve para quem.
